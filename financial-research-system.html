Verifiable Multi-Agent Financial Research System

A public, eval-driven agentic AI project that produces source-backed financial research memos with bull/bear debate, contradiction detection, thesis-drift tracking, and evidence-span citation validation.

Problem

LLM financial research can sound polished while hiding unsupported claims, stale evidence, arithmetic mistakes, or prompt-injected news content. The goal was not to create a trading bot; it was to build a research system where every factual claim could be traced, checked, and improved through evals.

Architecture

The system uses a LangGraph StateGraph with parallel evidence streams for market data, SEC filings, news, and quant data. A supervisor routes query types, bull and bear agents synthesize opposing cases, contradiction detectors compare evidence and claims, and a verifier can reroute failed claims back to source agents before a moderator assembles the final memo.

Evaluation Methodology

Every change is scored with an 8-metric LLM-as-judge rubric tracked in MLflow and DagsHub. The quick eval set uses AAPL and RIVN so each prompt change is tested against different financial profiles under same-model isolation. A targeted improvement pass moved the overall score from 2.21 to 3.29, a roughly 49% lift.

Reliability And Guardrails

The citation system uses source references tied to evidence spans instead of loose numeric citations. Agents cannot invent sources because citations are validated against the evidence list. A verifier flags high-severity unsupported claims and reroutes to relevant source agents, capped at two attempts to avoid infinite loops.

Adversarial Testing

The adversarial harness injects poisoned news spans across explicit instruction, role hijack, misleading quotation, fabricated metric, and source-reference spoof categories. The primary metric is absorption rate, with diagnostics showing whether a case was caught by the verifier, agent refusal, or citation layer.

Tradeoffs

I kept the per-source agent layer because two source agents do real domain work, and a generic fetch abstraction would hide important differences. I also avoided a broad step framework for LLM calls because the shared code was small, while the eval lift came from targeted per-agent prompt changes.
